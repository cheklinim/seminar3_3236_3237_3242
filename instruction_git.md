# **Инструкция по работе с системой контроля версий Git**

![Эмблема Git](git.jpg)

Git (произносится «гит») — распределённая система управления версиями. Проект был создан Линусом Торвальдсом для управления разработкой ядра Linux, первая версия выпущена 7 апреля 2005 года. На сегодняшний день его поддерживает Джунио Хамано.

## Инициализация репозитория

Чтобы создать (инициализировать) новый репозиторий нужно в терминале ввести команду:

    git init

Репозиторий будет создан в той папке, из которой вызывалась команда

## Проверка состояния репозитория

Чтобы проверить текущее состояние репозитория нужно ввести в терминале команду:

    git status

## Добавление изменения к отслеживанию версионности

Чтобы добавить сделанное изменение к отслеживанию (поместить в индекс) нужно ввести команду:

    git add <имя файла>

где вместо <имя файла> вводится путь к файлу относительно расположения репозитория.

## Фиксация изменений

Чтобы зафиксировать изменение используется команда:

    git commit

В таком случае откроется окно для ввоба краткого описания сделанных изменений.

Чтобы сделать это одновременно с фиксацией используется команда:

    git commit -m "комментарий"

## Просмотр истории изменений

Чтобы посмотреть историю изменений используется комада

    git log

Для просмотра изменений с выводом одного коммита в одну строку используется команда

    git log --oneline

Для просмотра всех имеющихся коммитов используется команда

    git log --all

Для просмотра лога с графическим изображением веток используется команда

    git log --graph

Все указанные флаги могут использоваться вместе:

    git log --all --oneline --graph

## Просмотр различий между изменениями

Для просмотра отличий между текущим состоянием репозитория и последним сохраненным изменением используется команда

    git diff

Можно также посмотреть разницу между любыми двуми коммитами. Для этого используется команда

    git diff <хэш1> <хэш2>

## Переключение на нужное изменение

Чтобы переключиться на нужный коммит используется команда

    git checkout <хэш>

## Ветки в Git

### Создание новой ветки

Чтобы создать новую ветку используется команда

    git branch <имя_ветки>

### Просмотр всех веток

Чтобы посмотреть какие ветки существуют и на какой мы находимся используется команда

    git branch

### Переключение между ветками

Чтобы переключиться на другую ветку используется команда

    git checkout <имя_ветки>

### Слияние веток

Чтобы влить одну ветку в другую необходимо находясь в целевой ветке (КУДА будем делать слияние) выполнить команду

    git merge <имя_вливаемой_ветки>

### Конфликты при слиянии

Если одна и та же строка в разный версиях записана по разному возникнет конфликт.
Чистый гит автоматически сохраняет оба изменения, далее требуется вручную внести нужные правки и сделать коммит.

VSСode дает возможность выбрать какое изменение сохранить (входящее, существующее или оба).

### Удаление ветки

Чтобы удалить ветку, которая больше не нужно (например после слияния) используется команда

    git branch -d <имя_ветки>

## **Работа с удаленными репозиториями**
__GitHub (github.com)__ - популярный сервис, предоставляемый Microsoft, основаный на системе контроля версий Git, позволяющий работать с удаленными репозиториями.

<br>
Как это работает:

Даже не имея учётки на github можно открыть ссылку на репозиторий любой общедоступной учетной записи (а их большинство, т.к. закрытые репозитории - платные, соответственно github - один из крупнейших источников открытого кода в мире), и с помощью команды

* __git clone <ссылка на репозиторий>__ - _скопировать в текущую папку, директорию с доступный репозиторием._

<br>
Чтобы же выложить свой репозиторий, необходимо зарегистрировать учетку, после чего зайти в свои репозитории, создать новый, задав ему имя, затем сам github подскажет что нужно сделать. Нас интересует то что находится под строками:

"... or push an existing repository from the command line" (копируем и вводим в терминале локального репозитория):
 * __git remote add origin <ссылка на текущий, свежесозданный репозиторий>__ - _указывает git куда отправлять файлы_
 * __git branch -М main__ - _указывает какая основная ветка локального репозитория (по умолчанию git основной ветке дает имя **master**, соответственно слово **main** в команде нужно изменить на **master**)_
 * __git push -u origin main__ - _пушит (толкает, отправляет) локальную версию репозитория на github (**main**, соответственно так же нужно изменить на имя основной ветки). При первом использовании текущей связки потребуется авторизация! В дальнейшем после изменения проекта можно выполнять просто **git push**_

 После выгрузки и обновления страницы удаленного репозитория, все его файлы и последний коммит отобразятся на странице, там же будет ссылка для его выгрузки на другие ПК через **git clone**.

<br>
Внеся изменения в проект с помощью другого пк или самого github.com (а такая возможность реализована, и даже позволяет комитить изменения), на ранее синхронизированный репозиторий можно перенести все новые правки с помошью команды:
* __git pull__ - _(от слова "тянуть") - "стянет" актуальный репозиторий, выполнив, вместе с тем, **merge** (сливание) с локальным._
<br>  
 **Форк**

Одна из важнейших же функций github - возможность _**сделать "форк"**_ (копию чужого удаленного репозитория в свой, с сылкой на автора), _с целью дополнения и изменения проекта, и удаленной работы несколькими авторами над одним кодом._

Для этого необходимо зайти на страницу интересующего нас репозитория (например через поиск по автору). И нажать кнопку "Fork" (Вилка), после чего репозиторий скопируется в текущую авторизованную учетку. Обычно каждый проект имеет файл README (так заведено на github), в котором можно почитать, или записать, краткое (или не очень) описание по проекту. 

С помощью ссылки со страницы "форкнутого" репозитория и команды _**git clone**_, так же переносим его на любое устройство, для последующего редактирования. Внесение изменений в проект принято проводить только с помощью отдельных веток: создаем отдельную ветку, в ней дополняем код, коммитим изменения. После чего выполняем _**git push**_, находясь в редактируемой ветке. Git предложит более корректную команду для переноса на github только текущей ветки. Копируем её и выполняем:

* **git push --set-upstream origin <ваша ветка>**

После обновления страницы репозитория, вместе со всеми изменениями и последним коммитом, добавится свежая ветка, на которую сверху вверху проекта можно переключиться. Так же обычно после переноса автоматом засияет яркая кнопочка $\color{green}{"compare}$  $\color{green}{and}$  $\color{green}{pull}$ $\color{green}{request"}$, позволяющая сразу предложить изначальному автору репозитория твои правки с указанием новой ветки. В соеобразном итоговом "диалоге" можно описать и обсудить все изменения.

Если кнопка _"compare and pull request"_ почему-то не появилась, можно войти в то же меню нажав `"Contribute" -> "Open pull request"`.
